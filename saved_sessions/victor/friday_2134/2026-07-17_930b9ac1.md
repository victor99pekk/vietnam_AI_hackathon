## 🧠 What Is This Project?

This is a **Knowledge Graph Generator** built for the **Vietnam AI Challenge (Hackathon)**. It takes raw text files and automatically converts them into structured **knowledge graphs** — networks of entities (people, places, organizations) connected by relationships.

### The 5-Stage Pipeline

```
Raw Text → [1. Ingest & Clean] → [2. Dedup & Filter] → [3. Extract] → [4. Resolve] → [5. Build Graph & Export]
```

| Stage | What it does |
|---|---|
| **1. Ingest** | Loads `.txt`, `.json`, `.csv` files and normalizes whitespace/encoding |
| **2. Dedup & Quality** | Removes duplicate/low-quality documents (MinHash/SimHash/n-gram) |
| **3. Extract** | Uses spaCy NER to find entities + rule-based (or LLM) relation extraction → produces `(subject, predicate, object)` triples |
| **4. Resolve** | Merges duplicate entity mentions via embedding similarity |
| **5. Build & Export** | Constructs a directed graph, validates it against an ontology, exports to JSON/GraphML/Neo4j CSV/RDF |

### Example — `data/sample/`

You have two sample files:

- **`alan_turing.txt`** — Talks about Turing, London, Cambridge, Princeton, Bletchley Park, Enigma, etc.
- **`marie_curie.txt`** — Talks about Curie, Warsaw, polonium, radium, Nobel Prizes, etc.

The pipeline extracts entities like `PERSON` (Alan Turing, Marie Curie), `GPE` (London, Warsaw), `ORG` (University of Paris), and relations like `(Alan Turing, worked_at, Bletchley Park)`, then builds a knowledge graph.
